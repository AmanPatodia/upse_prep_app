# Step-by-step Development Roadmap

## Phase 1 (Days 1-5): Foundation

1. Finalize architecture and route map.
2. Set up Riverpod + GoRouter + Hive adapters.
3. Implement auth/session and encrypted token storage.
4. Build design system tokens and reusable widgets.

## Phase 2 (Days 6-10): Content Core

1. Subjects/chapters/topics/note editor + reader.
2. Bookmark/highlight/revision markers with local persistence.
3. PDF/image attachment rendering and caching.

## Phase 3 (Days 11-15): Practice Engine

1. MCQ question bank with filters and timed mode.
2. Mock test engine (timer, review mode, explanations).
3. Attempt history, accuracy and speed analytics.

## Phase 4 (Days 16-20): PYQ + Current Affairs

1. PYQ browser with tagging/filtering.
2. Daily and monthly current affairs modules.
3. Revise-later queue and offline sync.

## Phase 5 (Days 21-24): AI Automation

1. AI summary and note conversion service integration.
2. AI-generated MCQ and revision question workflows.
3. Topic detection and push updates into UI feed.

## Phase 6 (Days 25-27): Analytics + Planner

1. Dashboard charts and subject-wise strength model.
2. Revision pending and target tracker.
3. Study streak and productivity insights.

## Phase 7 (Days 28-30): Hardening

1. Full test suite and bug fixing.
2. Performance tuning and pagination.
3. Private release packaging and backup/sync verification.
